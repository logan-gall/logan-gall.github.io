Spatial Processing Unit

The TinySPU is a prototype Spatial Processing Unit (SPU) that demonstrates how core GIS and spatial AI operations can be implemented directly in hardware. Designed for the TinyTapeout project, it introduces a compact, 4-bit processor with a Spatial Instruction Set Architecture (SISA) tailored to geospatial workloads. This project showcases the potential for lightweight, low-power chips capable of executing essential spatial operations with minimal computational overhead.

  • Compact Spatial Instruction Set: Implements 16 custom opcodes for geospatial tasks, including focal mean/sum/max pooling, dot products, Manhattan Distance, reclassification, and vector area calculations.
  • Purpose-Built Architecture: Uses four 4-bit inputs (A, B, C, D) and two outputs (M, N), allowing spatial operations on raster rows or coordinate pairs with greater flexibility than traditional ALUs.
  • Optimized for Spatial Logic: Hardware-level design eliminates redundant calculations in spatial neighborhoods, improving energy efficiency and throughput.
  • Geospatial AI Capabilities: Supports fundamental operations used in geoAI, such as convolution-style pooling and dot product accumulation.
  • Open and Extensible: Released as an open-source project, with a test bench to validate functions and a demo bench to showcase classic GIS workflows using spatial hardware instructions.
  • TinyTapeout Integration: Developed for the TinyTapeout fabrication platform, offering a proof-of-concept for spatial ASICs and demonstrating GIS-on-a-chip as a viable path forward.

This project represents a foundational step toward domain-specific hardware for geospatial computing, with future applications in real-time GIS, satellite image processing, and embedded navigation systems.

Mini-Rex: Drone-Based Wireless Control Link

Codenamed Mini-Rex, this project is a full hardware/software pipeline designed to wirelessly transmit radio control signals over long distances with near-zero latency. It is built based on robust drone control link protocols ExpressLRS and CRSF. This project expands the availability of input devices, such as joysticks, game controllers, or sensor arrays, into a variety of output devices ranging from FPV drones to computer games to robotics. Mini Rex features a modular architecture, allowing flexibility to nearly any use-case requiring low-latency, long range, easily manageable control.

  • Versatile Input Integration: Interfaces with multiple input devices including game controllers, joysticks, and sensors.
  • Long-Range, Low-Latency Communication: Utilizes ExpressLRS and/or CRSF protocols to ensure rapid, reliable wireless signal transmission.
  • Modular Design: Separates input processing and output interpretation, allowing users to easily customize the system to suit various applications.
  • Flexible Connectivity Options: Supports both USB and FTDI serial connections for transmitter modules, accommodating a wide range of hardware.
  • Dual Output Modes: Enables both emulated HID gamepad signals and PWM outputs, broadening the scope of potential end devices.
  • Low Cost: Utilizes radio control hardware standard in the FPV world and a Raspberry Pi Pico microcontroller, allowing for a full control setup for under $75.

WiFi-Based Indoor Location Tracking

This project explores the use of WiFi signals to determine indoor locations with high accuracy. Traditional GPS technology struggles in indoor environments due to signal interference, making alternative methods necessary. This study leverages WiFi signal strength (RSSI) and neural network models to predict a deviceâ€™s location within a room. The approach was tested in the Geocommons lab at the University of Minnesota, achieving precise localization with a mean error of about 1 foot. The results demonstrate the feasibility of WiFi-based indoor localization, providing a proof-of-concept for future applications in navigation, smart buildings, and presence detection.

  • WiFi-Based Positioning: Utilizes Received Signal Strength Indicator (RSSI) from multiple WiFi access points to estimate device location.
  • High Accuracy: Achieved an RÂ² score of 0.97 in predicting indoor coordinates, with an average error of approximately 1 foot.
  • Neural Network Modeling: Developed multiple models, comparing simple distance estimation and full (X, Y) localization.
  • Spatiotemporal Analysis: Investigated how RSSI values change over distance and time, revealing variability in signal stability.
  • Interactive Visualization: Created an interactive web tool to explore model predictions and WiFi signal trends dynamically.
  • Future Expansion: Potential improvements include incorporating LSTM models for better temporal tracking, expanding to larger areas, and integrating additional sensors such as Bluetooth or accelerometers.

Project MÃ©lange: LSTM-Based Evapotranspiration Modeling

Project MÃ©lange was a machine learning research initiative that explored the use of LSTM (Long Short-Term Memory) models to estimate actual evapotranspiration (ETa). The project leveraged gridded climate datasets and knowledge-guided machine learning (KGML) techniques to improve prediction accuracy. The goal was to enhance spatial and temporal interpolation and extrapolation of ETa, which is critical for agricultural water management and environmental monitoring.

This project integrated multiple data sources, including Ameriflux eddy covariance stations, the National Solar Radiation Database (NSRDB), and the SSEBop dataset, to develop models capable of predicting evapotranspiration with improved accuracy and generalizability.

  • LSTM-Based Machine Learning: Applied deep learning techniques for time-series modeling of ETa based on weather variables.
  • Knowledge-Guided Machine Learning (KGML): Incorporated domain knowledge from the Penman-Monteith equation to refine model predictions.
  • Gridded Data Processing: Handled large-scale climate datasets (NSRDB & SSEBop) for improved spatial coverage.
  • Custom Loss Function Development: Integrated physical constraints to enforce realistic predictions, preventing non-physical values like negative ETa.
  • High-Performance Computing: Utilized SLURM job scheduling on MSI Supercomputers for large-scale model training, requiring up to 275GB RAM.
  • Interpolation & Extrapolation Capabilities:
    • Spatial Interpolation: Filled gaps in gridded ETa estimates using climate data.
    • Temporal Interpolation: Predicted missing ETa values in time-series observations.
    • Spatial Extrapolation: Extended ETa estimates to new locations with minimal ground truth data.
    • Temporal Extrapolation: Forecasted future ETa based on past observations.
  • Alternative Model Exploration: Investigated ConvLSTM, diffusion models, and CNN architectures for potential improvements.
  • Scalability & Real-World Application: Designed to function with minimal weather station data, making it useful for agricultural and environmental applications.

This project demonstrated advanced geospatial AI techniques applied to climate and hydrological modeling, showcasing the potential of deep learning in environmental sciences.

Cloud-Driven Live Location Sharing

A live location sharing web application was developed that allows users to share their real-time location with friends or colleagues. The platform is designed to be lightweight, requiring no additional app downloads, and enables users to input their location manually or by clicking on the map. The system also provides dynamic route calculations to shared locations and real-time updates. Built using Google Cloud infrastructure, the application leverages Flask, Docker, Cloud SQL, and automated deployments to ensure a seamless and scalable experience.

  • Live Location Sharing: Users can enter their name, latitude, and longitude, or select a point on the map to share their location in real time.
  • Interactive Web Map: Built on an OpenStreetMap basemap, the interface allows users to visualize shared locations with clickable markers.
  • Route Calculation: The app dynamically finds and displays the quickest route from a central location (e.g., Blegen Hall at the University of Minnesota) to any shared point.
  • Cloud-Based Infrastructure: Utilizes Google Cloud SQL for storing locations, Flask for backend processing, and Docker for scalable deployment via Cloud Build & Run.
  • Real-Time Updates: Includes a proof-of-concept feature that allows users to ping their device to update their location automatically via the Pushover and Tasker apps.
  • Privacy-Focused: Users can share their location temporarily without requiring third-party apps that store personal data.
  • Potential Applications: Ideal for friends, event organizers, or companies needing a lightweight, temporary location-sharing tool for meetups, conferences, or team coordination.

Motion-Based Drone Controller

This project reimagines drone simulation control by using a real drone as the controller. By leveraging the drone's built-in inertial measurement unit (IMU) and a fast Python script, the system reads the drone's orientation and translates it into real-time game control inputs. This intuitive method bypasses the typical joystick interface, offering a more natural and engaging user experience for both learning to fly drones and exploring alternative control schemes.

  • Real-World Input for Simulation: Utilizes the droneâ€™s actual orientation as a control mechanism, providing a more intuitive interface than traditional joysticks.
  • IMU Integration: Extracts accelerometer and gyroscope data to determine the droneâ€™s direction and movement.
  • Multi-Wii Serial Protocol: Communicates directly with the drone using a protocol originally developed for Nintendo Wii controllers.
  • XBox Controller Emulation: Converts real-time drone orientation into standardized game inputs by emulating an XBox controller.
  • Enhanced User Experience: Offers an innovative approach that could be extended to advanced input devices and interactive educational tools.
  • Hybrid Control Scheme: Implements additional throttle control using buttons on an XBox adaptive controller, addressing the IMU's limitations in measuring relative motion.
